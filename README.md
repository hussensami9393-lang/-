# بوت تذكير مريم بالأوراق الحكومية

بوت تيليجرام عربي يساعدك على حفظ مواعيد تقديم الأوراق المهمة للمدير، مع تحديد اليوم والساعة والدقيقة والثانية. البوت يحتوي على أزرار سهلة لإضافة التذكيرات، عرضها، وحذفها، ويرسل رسالة تنبيه واضحة في الموعد المحدد.

## المميزات

- واجهة عربية بأزرار داخل تيليجرام.
- إضافة تذكير بعنوان وتفاصيل الأوراق المطلوبة.
- تحديد الموعد بصيغة تشمل الثواني مثل: `2026-06-15 09:30:00`.
- حفظ التذكيرات في قاعدة بيانات SQLite.
- فحص التذكيرات كل ثانية وإرسال التنبيه فور حلول الموعد.
- جاهز للرفع على Fly.io باستخدام Docker.
- يحتوي على ملفات بديلة لـ Render و Railway.

## 1. إنشاء بوت تيليجرام والحصول على التوكن

افتحي تيليجرام وابحثي عن `@BotFather`، ثم أرسلي الأمر `/newbot`. سيطلب منك اسم البوت واسم المستخدم. بعد الانتهاء سيعطيك توكن يشبه هذا الشكل:

`123456789:ABCDEF_your_token_here`

احتفظي بالتوكن لأنه سيستخدم في متغير البيئة `BOT_TOKEN`.

## 2. التشغيل المحلي على جهازك

ثبتي Python 3.11 أو أحدث، ثم نفذي الأوامر التالية داخل مجلد المشروع:

```bash
pip install -r requirements.txt
```

ثم شغلي البوت بهذه الطريقة، مع استبدال النص بتوكن بوتك الحقيقي:

```bash
export BOT_TOKEN="ضع_توكن_البوت_هنا"
export TIMEZONE="Asia/Riyadh"
python bot.py
```

بعد التشغيل افتحي البوت في تيليجرام واضغطي `/start`.

## 3. رفع البوت على Fly.io

> ملاحظة: كتبتِ fiy.io، وغالبًا المقصود هو Fly.io. هذا المشروع جاهز لـ Fly.io.

أولًا ثبتي أداة Fly CLI من موقع Fly.io، ثم سجلي الدخول:

```bash
fly auth login
```

داخل مجلد المشروع نفذي:

```bash
fly launch --no-deploy
```

إذا سألك عن اسم التطبيق يمكنك استخدام:

```text
maryam-reminder-bot
```

لإنشاء مساحة تخزين حتى لا تضيع قاعدة البيانات بعد إعادة التشغيل:

```bash
fly volumes create maryam_data --size 1 --region ams
```

أضيفي توكن البوت كـ secret:

```bash
fly secrets set BOT_TOKEN="ضع_توكن_البوت_هنا"
```

ثم ارفعي التطبيق:

```bash
fly deploy
```

لمشاهدة السجلات والتأكد أن البوت يعمل:

```bash
fly logs
```

## 4. الرفع على Render كبديل

ارفعي المشروع إلى GitHub، ثم ادخلي Render وأنشئي خدمة جديدة من نوع Worker. اختاري Docker، ثم أضيفي متغيرات البيئة التالية:

```text
BOT_TOKEN=توكن_البوت
TIMEZONE=Asia/Riyadh
DB_PATH=reminders.db
```

ملف `render.yaml` موجود في المشروع لتسهيل الإعداد.

## 5. الرفع على Railway كبديل

ارفعي المشروع إلى GitHub ثم اربطيه مع Railway. سيقرأ Railway ملف `railway.json` ويستخدم Dockerfile. أضيفي متغيرات البيئة:

```text
BOT_TOKEN=توكن_البوت
TIMEZONE=Asia/Riyadh
DB_PATH=reminders.db
```

## 6. صيغة التاريخ داخل البوت

عند إضافة تذكير، اكتبي التاريخ والوقت بهذه الصيغة:

```text
2026-06-15 09:30:00
```

وتعني: سنة-شهر-يوم ساعة:دقيقة:ثانية.

يمكن أيضًا استخدام:

```text
15-06-2026 09:30:00
```

## 7. ملاحظات مهمة

البوت يعمل بطريقة polling، لذلك لا يحتاج webhook ولا دومين. يجب فقط أن تكون الاستضافة قادرة على تشغيل عملية Python بشكل مستمر. لو استخدمتِ استضافة مجانية، قد تنام الخدمة أحيانًا حسب سياسة المنصة، لذلك Fly.io أو Railway أو Render Worker مناسبين أكثر من الاستضافات التي توقف العمليات الخلفية بسرعة.

## 8. ملفات المشروع

- `bot.py`: كود البوت الرئيسي.
- `requirements.txt`: مكتبات Python المطلوبة.
- `Dockerfile`: تشغيل البوت داخل Docker.
- `fly.toml`: إعداد Fly.io.
- `render.yaml`: إعداد Render.
- `railway.json`: إعداد Railway.
- `.env.example`: مثال لمتغيرات البيئة.

## 9. رسالة التذكير

عند حلول الموعد سيرسل البوت رسالة مثل:

```text
🚨 تذكير مهم لمريم 🚨
📣 لا تنسي تقديم الأوراق في موعدها حتى تحفظي حقك في الترقية.
📌 العنوان
📄 تفاصيل الأوراق
📅 الموعد المحدد
✅ جهزي الأوراق وقدميها للمدير في الوقت المحدد.
```
